# cns-wc2026cup-lab

本仓库用于归档和发布多个独立 HTML 页面。这些页面可能包括信息展示、工具界面、实验性小应用或静态演示内容。

## 项目简介

cns-wc2026cup-lab 是一个以 HTML 页面为主要内容的归档仓库。仓库中的页面彼此独立，不依赖特定后端服务，可直接在浏览器中打开使用。适用于快速分享原型、静态页面存档或轻量级前端演示。

## 目录说明

```
.
├── index.html          # 仓库主页（可选，用于导航）
├── pages/              # 存放各独立 HTML 页面
│   ├── example1.html
│   ├── example2.html
│   └── ...
└── assets/             # 公共资源（如 CSS、JS、图片等）
    ├── style.css
    └── script.js
```

- `pages/` 目录下存放所有独立页面文件，每个文件为一个完整可用的 HTML 页面。
- `assets/` 目录存放页面共享的样式、脚本和媒体资源，便于复用和更新。

## 页面归档说明

- 每个页面均为独立 HTML 文件，可直接双击或在服务器环境下访问。
- 页面间无强依赖关系，可单独更新或删除。
- 归档内容可能包含静态信息展示、工具界面、交互原型等，不涉及具体业务域名或网站。
- 页面中的链接、图片等资源均使用相对路径或 CDN 公共资源，避免依赖特定域名。

## 维护说明

- 新增页面：在 `pages/` 目录下创建新的 HTML 文件，并确保引用资源路径正确。
- 更新页面：直接修改对应 HTML 文件内容，注意保持页面完整性和兼容性。
- 删除页面：从 `pages/` 目录中移除文件，并在仓库主页（如有）更新导航链接。
- 公共资源：如需修改全局样式或脚本，请在 `assets/` 目录中更新，并检查各页面引用是否同步。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 直接在浏览器中打开 `pages/` 下的任意 HTML 文件。
3. 或通过 GitHub Pages 等服务部署后在线访问。

## 许可

本仓库内容仅供学习、归档与演示使用。如引用第三方资源，请遵守其相应许可协议。
